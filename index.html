TOPIC

GIT adalah sebuah tools bagi para programmer dan developer yang berfungsi sebagai control system untuk menjalankan proyek pengembangan software. GIT adalah singkatan dari Group Inclusive Tour. Tujuan penggunaan GIT yakni untuk mengelola versi source code program dengan menentukan baris serta kode yang akan ditambahkan atau diganti. Git adalah sebuah version control system terbuka yang dikembangkan oleh Linus Torvalds pada tahun 2005. Linus Torvalds juga merupakan pengembang sistem kernel dari sistem operasi Linux. Saat ini, Git merupakan version control system paling populer digunakan. Ada ratusan proyek software yang mengandalkan Git sebagai version control system mereka termasuk proyek komersial serta open source.

FUNCTION

Fungsi utama dari git adalah untuk mengatur versi dari sebuah source code program dengan cara memberikan tanda baris dan code yang akan ditambah atau diganti.

1. Akan sangat memudahkan para developer untuk dapat berpartisipasi di dalam sebuah pengembangan project yang dilakukan oleh orang-orang. 

2. Dapat memudahkan dalam melakukan sebuah dokumentasi project, baik dari sisi source code nya ataupun perubahan fungsi.

3. Membantu Organisir Membantu mengorganisasi adalah salah satu fungsi Git berikutnya. Dengan Git, kamu bisa menyimpan proyek ke folder dalam bentuk, v1, v2, v3 dan memiliki satu proyek dengan menggunakan database khusus yang berisi semua versi file.

4. Sebagai Platform Fleksibilitas Git adalah layanan hosting terlengkap di mana di antaranya yaitu Gitlab, GitHub, Bitbucket, dan SourceForge. Jadi, salah satu fungsi Git adalah sebagai platform fleksibilitas karena dapat digunakan sebagai solusi untuk hosting pada semua proyek.

5. Menjadi backup Fungsi Git yang terakhir adalah menjadi backup. Artinya, jika terjadi kesalahan atau masalah dalam mengembangkan versi terbaru, Git dapat dengan mudah mengembalikan ke dalam versi sebelumnya.

TOOLS

- Git Command-Line Interface (CLI): Git memiliki antarmuka baris perintah yang kuat yang memungkinkan pengguna untuk menjalankan perintah-perintah Git langsung dari terminal atau prompt perintah.

- Git GUI: Ada sejumlah antarmuka pengguna grafis (GUI) untuk Git yang membuatnya lebih mudah digunakan bagi mereka yang tidak terbiasa dengan antarmuka baris perintah. Beberapa contoh termasuk GitKraken, Sourcetree, dan GitHub Desktop.

- Git Clients: Ini adalah perangkat lunak klien yang memfasilitasi penggunaan Git. Beberapa di antaranya adalah Git for Windows, Git for macOS, dan Git for Linux. Mereka menyediakan instalasi Git dan seringkali GUI sederhana untuk menjalankan perintah Git.

- GitHub, GitLab, Bitbucket: Platform hosting repositori Git seperti GitHub, GitLab, dan Bitbucket adalah alat yang penting dalam kolaborasi pengembangan perangkat lunak. Mereka memberikan repositori Git yang dapat diakses dan dikelola secara daring, alat kolaborasi tim, pelacakan masalah, dan banyak lagi.

- Visual Studio Code (VS Code): Sebagai editor kode sumber terkenal, VS Code memiliki dukungan yang kuat untuk Git. Anda dapat mengelola dan menggabungkan perubahan langsung dari VS Code.

- Git Hooks: Git memiliki mekanisme yang disebut "hooks" yang memungkinkan Anda menjalankan skrip otomatis sebelum atau setelah suatu peristiwa Git tertentu, seperti commit atau push. Ini berguna untuk menjalankan tugas-tugas seperti linting kode, menjalankan pengujian otomatis, dan lainnya.

- Gitignore: File .gitignore memungkinkan Anda mengabaikan file atau direktori tertentu dalam repositori Git, sehingga Anda dapat menghindari mengunggah file-file yang tidak diinginkan atau berkas sementara.

- Git LFS (Large File Storage): Git LFS adalah ekstensi Git yang memungkinkan Anda mengelola berkas besar, seperti gambar atau video, dengan lebih efisien dan menyimpannya di luar repositori utama.

- Git Submodules: Git Submodules memungkinkan Anda menyertakan repositori Git lainnya dalam repositori Anda. Ini berguna ketika Anda ingin menggunakan proyek terpisah sebagai komponen dalam proyek Anda.

- Git Hosting Services: Selain platform besar seperti GitHub, GitLab, dan Bitbucket, ada berbagai penyedia hosting Git lainnya seperti GitHubs Enterprise, GitLab Self-Managed, dan GitBucket yang memungkinkan organisasi untuk menjalankan solusi Git sendiri.
